Local File Viewer Options

Mandatory Configuration for Local Image Viewer


For local image viewer to work, you must allow it to access local file URLs at chrome://extensions/



Note: you may have to click "details" for the exetnsion to find these settings...

Find a saved image file

Once Local Image Viewer has been is enabled, from a folder full of images:


Drag image into browser, or use Ctrl + O and select an image file.

You may also set your favorite web browser as the default handler for image file types.

Once Configured

Next and previous arrows will show up allowing you to switch to the next and previous images in the local directory.

Also you may use the arrow keys to navigate next → and previous ← while space bar controlls zoom.

The Enter key begins (or ends) autoplay aka slideshow mode - which requires fast mode is enabled above.

This allows your favorite web browser to operate as a functional replacement for the slow and buggy Windows Photo Viewer

** NOTE: only images are fully supported, level of support for other formats may vary greatly. Make sure the browser supports the format, and even then the next/previous function may not work with every format, especially incompatible with fast mode features.

*** NOTE: one directory at a time, if a directory malfunctions, switch to a different one then switch back. As of 2019 you MUST visit the directory in the web browser before you can browse it. "This extension will never cache more than one directory at a time" (even if may make some folder recursion UX available... so perhaps a future opt-in for multiple directory caches could exist).

Now you can use your favorite web browser to browse through directories of photos!
